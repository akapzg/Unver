# Unver 开发路线图

> 当前版本：v1.1.0 | 后端 7000+ 行 Rust | 前端 3600+ 行 JSX

---

## P0 — 安全加固

### 登录限速
- 登录失败 5 次后 15 分钟内拒绝同 IP 请求
- 可选验证码（如部署在公网）
- 前端显示剩余尝试次数

### IP 黑白名单（按代理规则）
- 每个子规则支持 `allow_ips` / `deny_ips`
- 支持 CIDR 格式（192.168.1.0/24）
- 白名单优先于黑名单

### Basic Auth（按代理规则）
- 每个子规则可选启用
- 用户名密码存数据库（bcrypt 哈希）
- 前端管理凭据弹窗

---

## P1 — 代理能力

### 请求/响应头修改
- 按子规则配置 `set_headers` / `remove_headers`
- 支持 `$host`、`$remote_addr` 等变量
- 预设常用模板（CORS、反代标准头）

### WebSocket 支持确认
- 验证 HTTP 反代是否已透传 Upgrade 头
- 没有就补上

### 自定义错误页
- 后端不可用时显示友好 HTML
- 支持按端口组配置
- 默认提供通用错误页

---

## P2 — 运维体验

### 历史流量图表
- Dashboard 增加 24h / 7d 流量趋势图
- 按端口组统计
- 数据存 SQLite，定期清理

### 告警通知
- 证书到期前 7 天 / 3 天 / 1 天 Telegram 通知
- 后端健康检查失败通知
- DDNS 更新失败通知

### HTTP-01 验证
- Let's Encrypt HTTP-01 挑战支持
- 适合没有 DNS API 权限的场景
- 前端验证方式下拉增加 HTTP-01 选项

---

## 暂缓

| 功能 | 原因 |
|------|------|
| 负载均衡 | 个人/小团队场景用不到多后端 |
| UDP 代理 | 需求量极小 |
| 插件系统 | 过早抽象，先打磨核心 |
